# Замыкания и область видимости

## Задача 4 — var/let в цикле с setTimeout ⭐⭐

Что выведет этот код? Почему?

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}

for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log(j), 0);
}
```

<details>
<summary>Ответ</summary>

**Ответ:**

```
// Первый цикл (var):
3
3
3

// Второй цикл (let):
0
1
2
```

**Почему так:**

`var` объявляет переменную в области видимости функции (или глобальной), а не блока. Все три колбэка `setTimeout` замыкаются на **одну и ту же** переменную `i`. К моменту выполнения колбэков цикл уже завершён и `i === 3`.

`let` создаёт новую привязку переменной `j` на каждую итерацию цикла. Каждый колбэк замыкается на **свою собственную** копию `j` с текущим значением.

Как исправить `var`-версию — три способа:

```js
// 1. Заменить на let
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}

// 2. IIFE — создать новую область видимости
for (var i = 0; i < 3; i++) {
  (function (i) {
    setTimeout(() => console.log(i), 0);
  })(i);
}

// 3. Передать значение через bind
for (var i = 0; i < 3; i++) {
  setTimeout(console.log.bind(null, i), 0);
}
```
