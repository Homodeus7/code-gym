# Блок 1 — JavaScript Core: Типы и приведение

## 🤔 Вопрос 1: Что такое NaN?

**Вопрос:** Что такое NaN? Как проверить значение на NaN?

**Ответ:**

`NaN` (Not a Number) — специальное числовое значение, которое возникает при невалидных математических операциях. Тип `NaN` — `"number"`.

Главная особенность: `NaN !== NaN` (единственное значение в JS, не равное самому себе).

```js
typeof NaN; // "number"
NaN === NaN; // false

// Способы проверки:
isNaN("abc"); // true  — но сначала приводит к числу! isNaN("") === false
Number.isNaN("abc"); // false — строгая проверка, без приведения типов
Number.isNaN(NaN); // true  — правильный способ

// Когда возникает NaN:
0 / 0; // NaN
parseInt("abc"); // NaN
Math.sqrt(-1); // NaN
undefined + 1; // NaN
```

Рекомендуется всегда использовать `Number.isNaN()` вместо глобального `isNaN()`.

## 🤔 Вопрос 2: Разница между null, undefined и NaN

**Вопрос:** В чём разница между `null`, `undefined` и `NaN`?

**Ответ:**

|         | `undefined`           | `null`                    | `NaN`                                  |
| ------- | --------------------- | ------------------------- | -------------------------------------- |
| Тип     | `"undefined"`         | `"object"`                | `"number"`                             |
| Смысл   | значение не присвоено | явное отсутствие значения | невалидный результат числовой операции |
| == null | `true`                | `true`                    | `false`                                |

```js
let x;
console.log(x); // undefined — переменная объявлена, но не инициализирована

let y = null; // null — явно «ничего»
console.log(typeof y); // "object" (историческая ошибка в JS)

let z = 0 / 0;
console.log(z); // NaN
console.log(typeof z); // "number"

// Сравнение:
null == undefined; // true  (нестрогое)
null === undefined; // false (строгое)
null == 0; // false
undefined == 0; // false

// Проверки:
x == null; // true для null и undefined (удобный паттерн)
Number.isNaN(z); // true
```

## 🤔 Вопрос 3: Как работает parseInt и parseFloat?

**Вопрос:** Как работают `parseInt` и `parseFloat`? Какие есть ловушки?

**Ответ:**

`parseInt(string, radix)` — парсит строку как целое число в указанной системе счисления (radix). Останавливается на первом нечисловом символе.

`parseFloat(string)` — парсит строку как число с плавающей точкой.

```js
parseInt("42px"); // 42 — берёт цифры до первого нечислового символа
parseInt("px42"); // NaN — начинается с нечислового символа
parseInt("10", 2); // 2 — парсит "10" как двоичное число
parseInt("010"); // 10 (в современных движках), но в старых — 8 (восьмеричное!)
parseInt(0.000005); // 0 — сначала toString(): "0.000005" → 0
parseInt(0.0000005); // 5 — toString(): "5e-7" → берёт "5" !

parseFloat("3.14abc"); // 3.14
parseFloat("abc"); // NaN

// Главная ловушка — всегда указывай radix:
parseInt("08", 10); // 8 (без radix в старых движках было бы 0)

// Альтернатива без ловушек:
Number("42px") + // NaN — строже, не прощает мусор
  "42"; // 42 — быстрое приведение к числу
```

## 🤔 Вопрос 4: Что такое type coercion?

**Вопрос:** Что такое type coercion? Примеры `==` vs `===`.

**Ответ:**

Type coercion (приведение типов) — автоматическое преобразование значений из одного типа в другой. JS делает это неявно во многих операциях.

`==` (нестрогое равенство) — выполняет приведение типов перед сравнением.
`===` (строгое равенство) — сравнивает без приведения, тип и значение должны совпадать.

```js
// == с приведением типов:
0 == "0"        // true  — строка "0" → число 0
0 == false      // true  — false → 0
"" == false     // true  — оба → 0
null == undefined // true — особый случай
null == 0       // false — null приводится только к undefined

// === без приведения:
0 === "0"       // false
0 === false     // false
null === undefined // false

// Неочевидный coercion в арифметике:
1 + "2"         // "12" — число → строка (конкатенация)
1 - "2"         // -1  — строка → число (вычитание)
true + true     // 2   — boolean → число
[] + []         // ""  — массивы → строки
[] + {}         // "[object Object]"
{} + []         // 0   — {} как блок, +[] = 0
```

Правило: используй `===` по умолчанию, `==` только для `value == null` (ловит `null` и `undefined` одновременно).

## 🤔 Вопрос 5: Truthy/falsy значения

**Вопрос:** Что такое truthy и falsy значения?

**Ответ:**

Falsy — значения, которые приводятся к `false` в булевом контексте. Их ровно 8:

```js
// Все falsy значения:
false
0
-0
0n          // BigInt нуль
""          // пустая строка
null
undefined
NaN

// Всё остальное — truthy, включая:
"0"         // true — непустая строка
"false"     // true
[]          // true — пустой массив
{}          // true — пустой объект
-1          // true
Infinity    // true
```

```js
// Практика:
if ([]) console.log("truthy"); // выведет — массив всегда truthy!
if ({}) console.log("truthy"); // выведет — объект всегда truthy!

// Частая ошибка:
const arr = [];
if (arr.length) {
  /* безопаснее, чем if (arr) */
}

// Boolean coercion:
Boolean(0); // false
Boolean(""); // false
Boolean([]); // true
!!null; // false — двойное отрицание как приведение к boolean
```
