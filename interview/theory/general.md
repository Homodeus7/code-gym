# Блок 7 — Общая грамотность

## 🤔 Вопрос 27: Что такое REST?

**Вопрос:** Что такое REST API? Основные принципы.

**Ответ:**

REST (Representational State Transfer) — архитектурный стиль для построения API. Не протокол, а набор ограничений.

**6 принципов REST:**
1. **Client-Server** — разделение UI и данных
2. **Stateless** — каждый запрос самодостаточен, сервер не хранит состояние клиента
3. **Cacheable** — ответы должны указывать, можно ли их кэшировать
4. **Uniform Interface** — единый интерфейс (URI + методы + статусы)
5. **Layered System** — клиент не знает, общается с сервером или proxy
6. **Code on Demand** (опционально) — сервер может отдавать исполняемый код

**Ресурсно-ориентированные URL:**
```
GET    /users          → список пользователей
GET    /users/42       → пользователь с id=42
POST   /users          → создать
PUT    /users/42       → обновить
DELETE /users/42       → удалить
GET    /users/42/posts → посты пользователя 42
```

## 🤔 Вопрос 28: Что такое XSS и CSRF?

**Вопрос:** Что такое XSS и CSRF атаки? Как защититься?

**Ответ:**

**XSS (Cross-Site Scripting)** — внедрение вредоносного JS в страницу:

```html
<!-- Уязвимость: вставка пользовательских данных без экранирования -->
<div>Привет, {{username}}</div>
<!-- Если username = <script>document.cookie → отправить на сервер атакующего</script> -->
```

Защита:
- Экранировать HTML-сущности (`<` → `&lt;`)
- `Content-Security-Policy` заголовок
- `HttpOnly` для важных cookie (JS не прочитает)
- Sanitize library для rich-text

**CSRF (Cross-Site Request Forgery)** — заставить браузер пользователя сделать запрос без его ведома:

```html
<!-- Страница атакующего: -->
<img src="https://bank.com/transfer?to=hacker&amount=1000" />
<!-- Браузер автоматически отправит cookie пользователя! -->
```

Защита:
- `SameSite=Strict` или `SameSite=Lax` для cookie — не отправлять с другого сайта
- CSRF-токен — уникальный токен в форме, проверяемый сервером
- Проверять заголовок `Origin`/`Referer`
